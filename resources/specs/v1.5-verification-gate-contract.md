# v1.5 Verification Gate Contract

## Scope
- Issue: `#170`
- Purpose: execute a deterministic milestone verification matrix and publish an explicit launch recommendation (`GO` or `NO-GO`) with auditable evidence.

## Canonical Inputs
1. Milestone child issue state snapshot (`#163`-`#170` under parent `#162`).
2. Deterministic test matrix command set:
   - ingestion + validation + inference + refresh + dashboard + fixture suites
   - console/kernel + semantic refresh command wiring suites
3. Fixture-pack determinism check:
   - `fixture:pack:refresh` hash on `tests/fixtures/scenarios`

## Canonical Outputs
1. Verification report:
   - `docs/plans/v1.5-verification-gate-report.md`
2. Explicit recommendation:
   - `GO` when required suites are green and blocking child issues are closed
   - `NO-GO` otherwise
3. Residual risk and follow-up section with concrete blocking issues.

## Required Matrix Commands
1. Ingestion matrix:
   - `./vendor/bin/phpunit --configuration phpunit.xml.dist packages/cli/tests/Unit/Ingestion/IngestionEnvelopeNormalizerTest.php packages/cli/tests/Unit/Ingestion/SchemaValidatorTest.php packages/cli/tests/Unit/Ingestion/SchemaDiagnosticEmitterTest.php packages/cli/tests/Unit/Ingestion/ValidationGateValidatorTest.php packages/cli/tests/Unit/Ingestion/ValidationDiagnosticEmitterTest.php packages/cli/tests/Unit/Ingestion/RelationshipInferenceEngineTest.php packages/cli/tests/Unit/Ingestion/SemanticRefreshTriggerPlannerTest.php packages/cli/tests/Unit/Command/IngestRunCommandTest.php packages/cli/tests/Unit/Command/IngestDashboardCommandTest.php packages/cli/tests/Unit/Command/IngestionFixturePackRegressionTest.php packages/cli/tests/Unit/Command/FixturePackRefreshCommandTest.php`
2. Kernel/wiring matrix:
   - `./vendor/bin/phpunit --configuration phpunit.xml.dist packages/foundation/tests/Unit/Kernel/ConsoleKernelTest.php packages/cli/tests/Unit/Command/SemanticRefreshCommandTest.php`
3. Fixture-pack hash check:
   - `php bin/waaseyaa fixture:pack:refresh --input-dir tests/fixtures/scenarios --json`

## Determinism Rules
- Commands and path inputs are fixed in report.
- Issue-state snapshot is captured in report at gate execution time.
- Fixture-pack hash must be stable across repeated runs.
- Report content is append-only for v1.5 auditability.
