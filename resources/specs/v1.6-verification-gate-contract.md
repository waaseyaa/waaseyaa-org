# v1.6 Verification Gate Contract

## Scope
- Issue: `#179`
- Purpose: execute the full v1.6 verification matrix (adapters, connectors, dedupe, merge, refresh, fixtures) and publish an explicit launch recommendation (`GO` or `NO-GO`) with auditable evidence; summarize residual risks and post-v1.6 follow-ups.

## Canonical Inputs
1. Milestone child issue state snapshot (`#173`–`#179` under parent `#172`).
2. Deterministic test matrix command set:
   - ingestion + adapter + connectors + dedupe + merge + refresh + dashboard + fixture suites
   - console/kernel + semantic refresh command wiring suites
3. Fixture-pack determinism check:
   - `fixture:pack:refresh` hash on `tests/fixtures/scenarios`

## Canonical Outputs
1. Verification report:
   - `docs/plans/v1.6-verification-gate-report.md`
2. Explicit recommendation:
   - `GO` when required suites are green and blocking child issues are closed
   - `NO-GO` otherwise
3. Residual risks and post-v1.6 follow-up section.
